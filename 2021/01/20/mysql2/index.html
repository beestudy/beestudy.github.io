权限

create database ruozedata; #新建数据库
grant all privileges on ruozedata.* to jepson@’%’ identified by ‘ruozedata’; #赋予权限
flush privileges; #刷新权限
对于mysql的用户操作，比如权限相关的 ，最后一步必须执行刷新权限。
%代表了 任意的客户端的IP地址 都被允许使用jepson用户来远程访问
grant all privileges on ruozedata.* to jepson@’16.2.3.%’ identified by ‘ruozedata’;
16.2.3这个网段的所有ip都允许有权限去访问，包括16.2.3.1和16.2.3.2等等

字段类型

数值类型

整型int

tinyint、smallint、mediumint、int/integer、bigint
1 2 3 4 8
特点：
①都可以设置无符号和有符号，默认有符号，通过unsigned设置无符号
②如果超出了范围，会报out or range异常，插入临界值
③长度可以不指定，默认会有一个长度
长度代表显示的最大宽度，如果不够则左边用0填充，但需要搭配zerofill，并且默认变为无符号整型
long 长整型

浮点型

定点数：decimal(M,D)
浮点数:
float(M,D) 4 单精度
double(M,D) 8 双精度
特点：
①M代表整数部位+小数部位的个数，D代表小数部位
②如果超出范围，则报out or range异常，并且插入临界值
③M和D都可以省略，但对于定点数，M默认为10，D默认为0
④如果精度要求较高，则优先考虑使用定点数 钱、金额最好用decimal

字符串

char 字节 定长0-255长度 自动补齐
varchar 字符串 变长0-65535字节
char、varchar、binary、varbinary、enum、set、text、blob

char：固定长度的字符，写法为char(M)，最大长度不能超过M，其中M可以省略，默认为1
varchar：可变长度的字符，写法为varchar(M)，最大长度不能超过M，其中M不可以省略

日期

date 日期 YYYY-MM-DD
time 时间 HH:MM:SS
datetime 年月日时分秒 8字节
timestamp 年月日时分秒 4字节 比较容易受时区、语法模式、版本的影响，更能反映当前时区的真实时间

SQL的语言分类

DQL（Data Query Language）：数据查询语言
select
DML(Data Manipulate Language):数据操作语言
insert 、update、delete
DDL（Data Define Languge）：数据定义语言
create、drop、alter
TCL（Transaction Control Language）：事务控制语言
commit、rollback

建表规范

规范:
1.表名称 字段名称 不要写中文 尽量不要汉语拼音
2.统一风格:
若已经存在表结构设计，风格比如是create_user，就按照此风格
若是新项目，可以自行主导风格
订单号：
order
orderno
orderid

创建时间:
ctime
cretime
createtime
cre_time
create_time
选择好风格 不能随意变动
3.第一个字段必须是自增长字段 主键，且 无业务意义 架构设计默认规则
4.一张表只有一个主键 id，业务字段需要唯一的话，就使用唯一约束来保证。

5.后面四个字段 务必加上
create_user create_time update_user update_time
6.业务字段 注释务必要加上comment ‘xxx’
如
name varchar(255) comment ‘姓名’,
age int(3) comment ‘年龄’,
